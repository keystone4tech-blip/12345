## 🛠 Инструкция по управлению проектом

### 1. Запуск Бота (через Docker)
Бот и его база данных работают в контейнерах. Для управления используйте терминал в папке `/bot`:

*   🚀 **Запустить**: `docker compose up -d`
*   🛑 **Остановить**: `docker compose down`
*   📜 **Посмотреть логи**: `docker compose logs -f bot`
*   🔄 **Пересобрать (при изменении кода)**: `docker compose up -d --build`

### 2. Запуск Личного Кабинета (локально)
Кабинет запускается через Node.js. Для управления используйте терминал в папке `/cabinet`:

*   📦 **Установить зависимости** (один раз): `npm install`
*   ▶️ **Запустить в режиме разработки**: `npm run dev`
*   🌐 **Открыть в браузере**: [http://localhost:3020](http://localhost:3020)

### 3. Связка компонентов
- **Бот** слушает API на порту `8080` (в Docker) или `8000` (локально).
- **Кабинет** настроен на `http://localhost:8000` для связи с базой данных через API бота.
- **Важно**: Для работы кабинета бот должен быть запущен!
- **🎨 Premium эмодзи**: Настраиваются в Админ-панели ➔ Настройки ➔ Конфигурация бота ➔ Интерфейс и брендинг ➔ Настройка Premium эмодзи. Чтобы захватить ID эмодзи, просто выберите его в сетке и отправьте нужный Premium-эмодзи боту в чат.

## 🛡 Контроль версий и Безопасность (Git)

### 🔒 Игнорируемые файлы (защита данных)
В проекте настроены файлы `.gitignore`, которые запрещают отправку конфиденциальных данных в публичный доступ:
*   **`.env`** — содержит токены бота, пароли БД и ключи API. **НИКОГДА** не удаляйте этот файл из игнора.
*   **`venv/`** — папка с библиотеками Python (занимает много места, не нужна в репозитории).
*   **`node_modules/`** — зависимости личного кабинета.
*   **`data/` & `*.db`** — файлы базы данных (содержат личные данные пользователей).
*   **`logs/`** — системные логи.

### 🚀 Автоматический деплой (CI/CD)
Проект настроен на автоматическое обновление сервера при каждом `git push` в ветку `main`.
- **Конфигурация**: `.github/workflows/deploy.yml`
- **Процесс**: GitHub подключается к вашему серверу через SSH → скачивает новый код → пересобирает Docker-контейнеры.

### 🔑 Что нужно настроить на GitHub:
Для работы деплоя добавьте в **Repository Secrets**:
1.  `SERVER_HOST` — IP вашего сервера.
2.  `SERVER_USER` — логин (обычно `root`).
3.  `SSH_PRIVATE_KEY` — содержимое вашего приватного SSH-ключа (позволяет GitHub войти на сервер без пароля).

### 📝 Как загружать обновления:
После внесения изменений на компьютере:
1. `git add .`
2. `git commit -m "Тип: Подробное описание изменений"`
   - **Правило**: Пишите содержательные коммиты (например: `feat: add premium emoji grid`, а не просто `update`).
   - Описывайте, какие модули затронуты и зачем сделаны изменения.
3. `git push origin main` — после этой команды сервер обновится сам в течение 1-2 минут.
